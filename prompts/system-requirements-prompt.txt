# Context

System requirements specify the functional and non-functional requirements that the system must meet.

# Naming & Syntax Rules

1. **Quoting:** If a name contains spaces or special characters (other than underscores), it MUST be enclosed in single quotes (e.g., 'System User').
2. **Identifier**: In particular, human readable identifiers which are modeled as short names of concerns and requirements, for example, must be encoded in single quotes if they contain dashes..
3. **Casing:** 
   - Definitions (part def, requirement def) must use **UpperCamelCase**.
   - Usages (part, requirement) must use **lowerCamelCase**.
4. **SysML Syntax:**
   - There is never a semicolon after a closing curly bracket "}}".
   - An short name in SysML v2 which is also used for concern or requirement identifier must be enclosed in "<>". If the identifier contains any other characters than letters and numbers, it must be enclosed in "<'identifier'>" instead of "<identifier>"

            subject system :  {name}_System_Package::{name}_System;
            #extendedRequirement requirement <!REQUIREMENT_IDENTIFIER!> !REQUIREMENT_NAME! {{

                attribute :>> priority = priority.medium;
                attribute :>> obligation = obligation.mandatory;
                attribute :>> stability = stability.stable;
                attribute :>> motivation = "!REQUIREMENT_MOTIVATION!";
                require constraint {{
                    doc /* !REQUIREMENT_TEXT! */
                }}
            }}
        }}
    }}

    Priority, obligation and stability are defined by PriorityKind, ObligationKind, and StabilityKind, which are defined in the SYSMOD library:

        enum def PriorityKind :> ScalarValues::Real {{
            none = 0.0;
            low = 0.25;
            medium = 0.5;
            high = 0.75;
            critical = 1.0;
        }}
        enum def ObligationKind {{
            none;
            mandatory;
            optional;
        }}
        enum def StabilityKind {{
            none;
            unstable;
            stable;
        }}
